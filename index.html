Taxonomy

Loop 위치로 나누는 coding agent 계층

IDE embedded

Cursor, Cline/Roo, Continue, Windsurf/Devin Desktop. Editor index와 diff UX가 강하지만 hidden context와 stale index가 위험하다.

CLI coding agent

OpenAI Codex CLI, Claude Code, Aider, Goose. Repo-local shell/test 재현성이 강하지만 sandbox와 approval mismatch가 중요하다.

Repo-local runner

SWE-agent, OpenHands, Agentless, Gajae-Code. issue → inspect → edit → test → retry 흐름과 benchmark/test oracle이 핵심이다.

Workflow orchestrator

LazyCodex/OmO, Gajae, Hermes delegation. planning, approval, evidence, continuation, team lanes를 강제하는 제어층이다.

Multi-agent/team

실제 orchestration은 durable queue/state/worktree/tmux/mailbox가 있어야 한다. 단순 role prompt와 구분한다.

Persistent runtime

Hermes, Devin-like long-running surfaces. repo session을 넘어 memory, gateway, cron, delegation으로 이어진다.

Common Architecture

공통 under-the-hood 구성요소

Model client / router
provider/model 선택, streaming, retries, token budget, task-specific routing. “agent 품질”은 모델만이 아니라 이 router와 prompt/tool layer의 결합이다.
Prompt / context assembly
system instructions, repo rules, active files, git diff, search results, memory, failing tests, tool outputs를 압축·선택·정렬한다. 가장 중요한 hidden decision point다.
Repo index / code graph
lexical search, semantic index, AST/symbol graph, repo map, LSP/MCP. stale index는 모델 문제가 아니라 context layer 실패다.
Planning / tool loop
plan 생성, tool 선택, observation 반영, 계획 수정. ReAct, Reflexion, plan-and-execute 패턴이 제품 워크플로우로 번역되는 지점이다.
Edit / execution
patch, diff, search/replace, whole-file rewrite, IDE write API, shell command, browser/MCP tool. over-editing과 patch failure가 여기서 발생한다.
Sandbox / approval
filesystem/network/shell 권한, ask/allow/deny, YOLO/auto mode, enterprise policy. safety UX가 아니라 core control plane이다.
Verification / evidence
tests, lint, build, benchmark, manual QA, reviewer agent, evidence ledger. “patch 작성”과 “동작 확인”을 분리한다.
Memory / state
session transcript, project memory, checkpoints, ledgers, skills, long-term user memory. continuity와 stale state risk가 동시에 생긴다.

Implementation Deep Dives

이번 UltraResearch의 주요 판정

Codex / Claude Code

  • CLI/runtime agent의 핵심은 sandbox, approval, project instructions, tool execution이다.
  • closed/internal context ranking은 문서화된 control surface와 분리해 낮은 confidence로 둔다.

Aider

  • RepoMap과 edit format이 공개되어 context compression과 patch strategy를 연구하기 좋다.
  • lint/test integration은 verification gate 연구에 유용하다.

OpenHands / SWE-agent / Agentless

  • runtime, sandbox, action/observation, benchmark harness가 agent 품질에 직접 관여한다.
  • Agentless는 모든 작업에 full autonomous loop가 필요한 것은 아님을 보여준다.

IDE-style agents

  • IDE index와 editor diff UX가 강점이다.
  • hidden context selection, stale index, approval fatigue가 주요 실패 원인이다.

LazyCodex / OmO

  • 로컬 확인상 Codex 위 lifecycle hooks, skills, MCP, ultrawork discipline을 얹는 harness다.
  • manifest 선언과 실제 runtime callable surface는 다음 단계에서 분리 검증해야 한다.

Gajae-Code / Hermes

  • Gajae는 interview/plan/ultragoal/team 중심의 runner로 보되 local gjc 실행은 미검증 caveat가 남는다.
  • Hermes는 repo editor라기보다 memory/skills/cron/gateway/delegation이 있는 persistent coordinator다.
Papers & Failure Patterns

논문과 실제 caveat를 architecture 원인으로 연결

SWE-bench 계열

repo-level patch evaluation을 가능하게 하지만 benchmark score를 제품 품질로 직접 읽으면 안 된다. incomplete tests와 semantic correctness gap이 남는다.

ReAct / Reflexion / Toolformer

reasoning-action loop, self-critique, tool use, planning 패턴의 이론적 기반이다. 특정 제품 구현의 정확성을 자동 보증하지는 않는다.

실패 패턴

context rot, over-editing, flaky tests, sandbox mismatch, stale index, hidden state drift, approval fatigue, benchmark overfitting을 architecture cause로 추적한다.
