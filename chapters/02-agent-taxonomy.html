Chapter 02

AI Agent / Harness / Workflow Orchestrator кө¬л¶„

мқҙм „мӨ‘мӢ¬ м§Ҳл¬ёлӢӨмқҢGajae-Code

Workflow Control Planeмқ„ л§җн•ҳл Өл©ҙ лЁјм Җ м„ё мёөмқ„ л¶„лҰ¬н•ҙм•ј н•ңлӢӨ. л§ҺмқҖ л¬ём„ңк°Җ вҖңagentвҖқ, вҖңrunnerвҖқ, вҖңharnessвҖқ, вҖңorchestratorвҖқ, вҖңworkflowвҖқлқјлҠ” лӢЁм–ҙлҘј м„һм–ҙ м“°м§Җл§Ң, мқҙ мұ…м—җм„ңлҠ” мқҳлҸ„м ҒмңјлЎң кө¬л¶„н•ңлӢӨ. кө¬л¶„мқҳ лӘ©м ҒмқҖ мҡ©м–ҙ мӢёмӣҖмқҙ м•„лӢҲлқј claimмқ„ м •нҷ•нһҲ нҸүк°Җн•ҳкё° мң„н•ҙм„ңлӢӨ. м–ҙл–Ө мӢңмҠӨн…ңмқҙ лӢЁмқј patch loopл§Ң м ңкіөн•ҳлҠ”лҚ° orchestratorлқјкі  л§җн•ҳл©ҙ кіјмһҘмқҙкі , м–ҙл–Ө мӢңмҠӨн…ңмқҙ workflow stateмҷҖ recoveryлҘј кҙҖлҰ¬н•ҳлҠ”лҚ° лӢЁмҲң agentлқјкі  л¶ҖлҘҙл©ҙ н•өмӢ¬мқ„ лҶ“м№ңлӢӨ.

м„ё мёөмқҳ working definition

мёөмӨ‘мӢ¬ лӢЁмң„н•өмӢ¬ мұ…мһ„лҢҖн‘ң м§Ҳл¬ё
AI Agentturn, message, tool call, patch, test runлӘ©н‘ңлҘј л°ӣм•„ contextлҘј л§Ңл“Өкі  tool/edit/test loopлЎң кІ°кіјл¬јмқ„ л§Ңл“ лӢӨ.мқҙ agentлҠ” н•„мҡ”н•ң нҢҢмқјмқ„ м°ҫкі  м•Ҳм „н•ҳкІҢ мҲҳм •н•ҳкі  кІҖмҰқн–ҲлҠ”к°Җ?
Harnesstask, environment, trajectory, score, oracleagentлҘј нҠ№м • нҷҳкІҪм—җ л„Јкі  мӢӨн–үВ·кҙҖм°°В·нҸүк°Җн•ңлӢӨ.мқҙ мӢӨн–үмқҖ мһ¬нҳ„ к°ҖлҠҘн•ҳл©° к°ҷмқҖ taskм—җм„ң л№„көҗ к°ҖлҠҘн•ңк°Җ?
Workflow Orchestrator / Control Planeworkflow, phase, state root, artifact, approval gate, worker, resume/recoveryagent/harnessлҘј нҸ¬н•Ён•ң мһ‘м—… нқҗлҰ„мқ„ м„Өкі„В·нҶөм ңВ·кё°лЎқВ·ліөкө¬н•ңлӢӨ.мқҙ мӢңмҠӨн…ңмқҖ кёҙ мһ‘м—…мқ„ лӢЁкі„нҷ”н•ҳкі  мҰқкұ°мҷҖ ліөкө¬ м§Җм җмқ„ мң м§Җн•ҳлҠ”к°Җ?

мқҙ м •мқҳлЎң ліҙл©ҙ Gajae-Code, Hermes, LazyCodex/OmOлҠ” лӘЁл‘җ мқјл°ҳ agent л№„көҗн‘ңмқҳ л§Ҳм§Җл§ү мӨ„м—җ лҶ“кё° м–ҙл өлӢӨ. м„ё мӢңмҠӨн…ң лӘЁл‘җ м–ҙл–Ө л°©мӢқмңјлЎңл“  agent мӢӨн–үмқҳ л°”к№ҘмӘҪм—җ workflow disciplineмқ„ м–№лҠ”лӢӨ. лӢӨл§Ң к°ҷмқҖ orchestratorлҠ” м•„лӢҲлӢӨ. Gajae-CodeлҠ” repo-local phase orchestrator, HermesлҠ” persistent runtime/meta-orchestrator, LazyCodex/OmOлҠ” Codex-embedded control overlayм—җ к°Җк№қлӢӨ.

л¶„м„қ м¶• 1 вҖ” phase model

Workflow OrchestratorлҠ” мһ‘м—…мқ„ phaseлЎң лӮҳлҲҲлӢӨ. лӢЁмқј agent loopлҠ” вҖңмҡ”мІӯ вҶ’ tool call вҶ’ patch вҶ’ test вҶ’ лӢөліҖвҖқмңјлЎң лҒқлӮ  мҲҳ мһҲлӢӨ. л°ҳл©ҙ orchestratorлҠ” interview, planning, approval, execution, review, checkpoint, handoff к°ҷмқҖ phaseлҘј л§Ңл“ лӢӨ. Gajae-Codeмқҳ deep-interview вҶ’ ralplan вҶ’ ultragoal вҶ’ teamмқҖ мқҙ м¶•м—җм„ң к°ҖмһҘ м„ лӘ…н•ҳлӢӨ. LazyCodex/OmOмқҳ $ulw-plan вҶ’ $start-work вҶ’ $ulw-loop кі„м—ҙлҸ„ phase modelмқ„ м§Җн–Ҙн•ңлӢӨ. HermesлҠ” кі м •лҗң coding phaseліҙлӢӨ slash commands, skills, cron, delegationмқ„ нҶөн•ҙ лӢӨм–‘н•ң workflowлҘј кө¬м„ұн•ҳлҠ” runtimeм—җ к°Җк№қлӢӨ.

л¶„м„қ м¶• 2 вҖ” state root

Control planeмқҖ stateк°Җ м–ҙл””м—җ лӮЁлҠ”м§ҖлҘј лӘ…нҷ•нһҲ н•ҙм•ј н•ңлӢӨ. agent transcriptл§Ң лӮЁмңјл©ҙ кёҙ мһ‘м—…мқҳ canonical stateлҘј м•Ңкё° м–ҙл өлӢӨ. Gajae-CodeлҠ” .gjc/_session-* м•„лһҳ specs, plans, ultragoal goals, ledger, team stateлҘј лӮЁкёҙлӢӨ. LazyCodex/OmOлҠ” .omo/... кі„м—ҙ project memory, workflow ledgers, Codex plugin cacheмҷҖ м—°кІ°лҗңлӢӨ. HermesлҠ” ~/.hermes/ м•„лһҳ sessions, memory, skills, logs, cron state, gateway stateлҘј к°–лҠ”лӢӨ. state rootк°Җ мһҲлӢӨлҠ” мӮ¬мӢӨл§ҢмңјлЎң orchestratorк°Җ лҗҳлҠ” кІғмқҖ м•„лӢҲм§Җл§Ң, state root м—Ҷмқҙ workflow orchestrationмқ„ мЈјмһҘн•ҳкё°лҠ” м–ҙл өлӢӨ.

л¶„м„қ м¶• 3 вҖ” artifact ledger

мўӢмқҖ orchestratorлҠ” вҖңмҷ„лЈҢн–ҲлӢӨвҖқлҠ” л§җліҙлӢӨ мӮ°м¶ңл¬ј ledgerлҘј лӮЁкёҙлӢӨ. Gajae-Code replayм—җм„ңлҠ” deep-interview spec, ralplan planner/final artifacts, pending approval, ultragoal goals/ledger, team dry-run stateк°Җ лӮЁм•ҳлӢӨ. LazyCodex/OmO docsлҠ” .omo/ulw-loop м•„лһҳ brief/goals/ledger к°ҷмқҖ кө¬мЎ°лҘј м„ӨлӘ…н•ңлӢӨ. HermesлҠ” tool results, trajectories, session history, scheduled job output, skills, memory updatesк°Җ runtime artifactк°Җ лҗңлӢӨ. artifact ledgerк°Җ мһҲм–ҙм•ј controllerлӮҳ мӮ¬лһҢ reviewerк°Җ claimмқ„ кІҖмҰқн•  мҲҳ мһҲлӢӨ.

л¶„м„қ м¶• 4 вҖ” approval/gate

Workflow Control PlaneмқҖ мң„н—ҳн•ң мӢӨн–үмқ„ gateлЎң нҶөм ңн•ңлӢӨ. лӢЁмқј agentлҸ„ command approvalмқ„ к°Җм§Ҳ мҲҳ мһҲм§Җл§Ң orchestratorмқҳ gateлҠ” phase м „нҷҳкіј м—°кІ°лҗңлӢӨ. кі„нҡҚмқҙ мҠ№мқёлҗҳкё° м „ мӢӨн–үн•ҳм§Җ м•ҠлҠ”лӢӨл“ м§Җ, review artifactк°Җ м—Ҷмңјл©ҙ лӢӨмқҢ goalмқ„ л§үлҠ”лӢӨл“ м§Җ, mutation opt-inмқҙ м—Ҷмңјл©ҙ MCP bridgeк°Җ fail-closedн•ҳлҠ” л°©мӢқмқҙлӢӨ. Gajae-Codeмқҳ pending approvalкіј review blockerлҠ” мқҙ м¶•мқҳ мӨ‘мҡ”н•ң мӢ нҳёлӢӨ. HermesлҠ” tool approval, gateway command approval, cron delivery, MCP mutation opt-in к°ҷмқҖ л„“мқҖ мҡҙмҳҒ gateлҘј к°–лҠ”лӢӨ. LazyCodex/OmOлҠ” Codex hook approvalкіј plugin capability approvalм—җ л¬¶мқёлӢӨ.

л¶„м„қ м¶• 5 вҖ” worker/process orchestration

OrchestratorлқјлҠ” л§җмқҖ worker coordinationмқ„ мҡ”кө¬н•ңлӢӨ. role promptлҘј м—¬лҹ¬ к°ң л§Ңл“ лӢӨкі  м¶©л¶„н•ҳм§Җ м•ҠлӢӨ. мӢӨм ңлЎңлҠ” worker identity, worktree/tmux/session, mailbox, task queue, heartbeat, cleanup, conflict policyк°Җ н•„мҡ”н•ҳлӢӨ. Gajae-Codeмқҳ team --dry-runмқҖ state layoutмқ„ ліҙм—¬мЈјм§Җл§Ң live tmux worker orchestrationмқҖ м•„м§Ғ лӢӨмқҢ кІҖмҰқмқҙлӢӨ. Hermesмқҳ delegate_task, background process, cron job, MCP/ACP bridgeлҠ” multi-worker control planeмңјлЎң ліј мҲҳ мһҲлӢӨ. LazyCodex/OmOлҠ” Codex subagent/hook lifecycleкіј м—°кІ°лҗҳм§Җл§Ң мӢӨм ң worker parityлҠ” лҚ” нҷ•мқён•ҙм•ј н•ңлӢӨ.

л¶„м„қ м¶• 6 вҖ” recovery/resume

кёҙ coding workflowлҠ” мӢӨнҢЁн•ңлӢӨ. л”°лқјм„ң ліөкө¬ м§Җм җмқҙ м—Ҷмңјл©ҙ demoлҠ” к°ҖлҠҘн•ҙлҸ„ мҡҙмҳҒмқҖ м–ҙл өлӢӨ. recoveryлҠ” лӢЁмҲңнһҲ мқҙм „ transcriptлҘј л¶Ҳлҹ¬мҳӨлҠ” кІғмқҙ м•„лӢҲлқј, нҳ„мһ¬ phase, pending artifact, blocked goal, worker state, approval stateлҘј мһ¬кө¬м„ұн•ҳлҠ” лҠҘл ҘмқҙлӢӨ. Gajae-CodeлҠ” .gjc stateк°Җ мқҙ к°ҖлҠҘм„ұмқ„ м—ҙм–ҙ мЈјм§Җл§Ң corruption/recovery replayк°Җ м•„м§Ғ н•„мҡ”н•ҳлӢӨ. HermesлҠ” session resume, memory, background process, cron output, skill persistenceлҘј нҶөн•ҙ session л°– runtimeмқ„ мң м§Җн•ңлӢӨ. LazyCodex/OmOлҠ” .omo ledgerк°Җ к·јкұ°к°Җ лҗҳм§Җл§Ң stale stateмҷҖ hook non-fireлҘј кІҖмҰқн•ҙм•ј н•ңлӢӨ.

л¶„м„қ м¶• 7 вҖ” evidence/replay boundary

л§Ҳм§Җл§ү м¶•мқҖ claimмқҳ мҰқкұ° кІҪкі„лӢӨ. source-confirmed claimмқҖ вҖңмҪ”л“ңм—җ мһҲлӢӨвҖқлҠ” лң»мқҙкі , runtime-confirmed claimмқҖ вҖңлӘ…л №мқҙ м§ҖкёҲ м„ұкіөн–ҲлӢӨвҖқлҠ” лң»мқҙл©°, artifact-backed claimмқҖ вҖңмӢӨн–ү нӣ„ нҷ•мқё к°ҖлҠҘн•ң durable artifactк°Җ лӮЁм•ҳлӢӨвҖқлҠ” лң»мқҙлӢӨ. manifest-confirmed claimмқҖ вҖңм„ м–ёлҗҳм–ҙ мһҲлӢӨвҖқлҠ” лң»мқҙм§Җл§Ң runtime callabilityлҘј ліҙмһҘн•ҳм§Җ м•ҠлҠ”лӢӨ. мқҙ кө¬л¶„мқҙ м—Ҷмңјл©ҙ LazyCodex/OmOмқҳ plugin manifestмҷҖ мӢӨм ң $... workflow мӢӨн–үмқ„ нҳјлҸҷн•ҳкұ°лӮҳ, Gajae-Codeмқҳ team dry-runкіј live teamмқ„ нҳјлҸҷн•ҳкұ°лӮҳ, Hermes docs мөңмӢ  кё°лҠҘкіј local installed runtimeмқ„ нҳјлҸҷн•ҳкІҢ лҗңлӢӨ.

мқҙ н”„л Ҳмһ„мқҳ кІ°лЎ 

мқҙ н”„л Ҳмһ„мқ„ м Ғмҡ©н•ҳл©ҙ мқҙ мұ…мқҳ кө¬мЎ°к°Җ мһҗм—°мҠӨлҹҪкІҢ м •лҰ¬лҗңлӢӨ. Chapter 03мқҖ phase/state/artifactк°Җ к°ҖмһҘ лӘ…нҷ•н•ң Gajae-CodeлҘј ліёлӢӨ.
